# 1. 产品介绍

<span id="u1RMf0D4"></span>
## 1.1 产品名称

OmniHuman1.5

<span id="vAFcDqR3"></span>
## 1.2 产品介绍

OmniHuman1.5（即梦同源数字人模型），该模型可根据用户上传的**单张图片+音频**，生成与图片对应的**视频效果**。支持输入任意画幅包含人物或其他主体（宠物、动漫等）的图片，结合音频，生成高质量的视频。

人物的情绪、动作与音频具有强关联性，支持通过提示词（prompt）对画面、动作、运镜进行调整。同时OmniHuman1.5对动漫、宠物等形象支持较好，允许指定讲话人/主体，可广泛应用于内容表达、唱歌和表演等场景。

相较于上一代模型，OmniHuman1.5 在**运动自然度和结构稳定性**提升明显，在**人物/主体的运动表现力和画面质量上更优。** 可以广泛应用于制作剧情对话、多人对话/对唱、商品交互、漫剧等内容。对比其他视频通用模型，OmniHuman 数字人大模型在**人物/主体的剧情演绎效果**上极具优势。

<span id="mKh93Vtj"></span>
## 1.3 产品优势


1. **节奏性表演：** 模型效果的灵活性延伸到了音乐领域，算法能够仅凭一张图像和一首歌曲，创造出富有情感的数字歌手。在我们的推理模块驱动下，动作不仅限于简单的口型同步，还包括自然的停顿和节奏变化，能够灵活应对从独唱抒情歌曲到欢快演唱会等各种风格。

2. **情感表演：** 仅通过一张图像和一段音频，模型就能让数字演员“活”起来。通过分析音频的情感内涵（无需文本提示），它能够生成引人入胜、富有电影感的表演，涵盖从激烈愤怒到真挚告白的完整戏剧表现力。

3. **上下文感知：** 我们的模型通过解析音频的语义上下文，超越了简单的口型同步和重复动作，使角色能够表现出真实的情感变化，并让动作与他们的言语和意图相匹配，仿佛他们拥有自己的意志。

4. **文本引导的多模态表现：** 算法精心设计的框架接受文本提示，并展示出卓越的提示遵循能力，从而能够精确控制物体生成、镜头运动和特定动作，同时保持完美的音频同步。

5. **单人/多人场景表演：** 模型的框架扩展到复杂的多人场景，它通过将单独的音轨路由到单个画面中正确的角色，来生成动态的表演，支持指定说话人或一起表演。

6. **支持多样化主体：** 模型展示了绝佳的鲁棒性，涵盖了令人难以置信的多样化主题，包括真实的动物、拟人化的角色和风格化的卡通

# 2. 提示词指南

<span id="nHylctuX"></span>
## **2.1 核心原则**


* **<ins>把编写提示词当成讲故事</ins>**，使用连贯自然的自然语言，尽可能减少孤立词汇的堆砌。

* 只描述动态的事件即可，图片中已经包含的静态特征（角色穿什么衣服、戴什么首饰等等）无需描述。

* 提示词需要遵循清晰、不矛盾、少否定的原则。使用具体而非抽象的描述，分步骤引导。


<span id="4KXbQdy5"></span>
## **2.2 最佳实践模板**

**推荐结构：** 镜头运动 + 说话角色情绪 + 说话状态（说话/哭泣/唱歌/...） + 具体动作 + （可选）背景事件/其他角色的动作

> **完整示例：** "镜头从侧面缓慢推进到正面中景。年轻女子坐在窗边，心情平静，轻轻端起咖啡杯，微笑着对镜头说话。旁边的男孩悄悄看着她，时不时转向镜头笑笑。背景中灯光柔和地闪烁。"

## 2.4 注意事项

<span id="jYtbZEoI"></span>
#### 推荐做法：


* 在提示词中将“角色xxx说话”“角色xxx唱歌”写出，有助于提高说话表现

* 视频清晰度高度依赖输入的图片！尽可能上传清晰度更高的图片

* 单次生成效果具随机性，复杂场景可多次抽卡尝试

* 输入提示词效果不佳，也可以尝试不输提示词看看


<span id="35etKgQF"></span>
#### 效果问题规避方法：


* 结构稳定性 15 秒以内较好，15 秒后可能出现衰退

* 人脸占比过小会偶现口型不动（角色不说话）的现象，尽可能规避超远景

* 人物长时间出镜再入镜后易出现一致性变化，设计提示词时尽量规避角色反复出镜的情况

* 极端姿势或角度效果可能不佳
